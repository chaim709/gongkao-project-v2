# 公考管理系统 V2 开发起点文档（2026-03-23）

> 目的：给后续 V2 开发一个单一入口，快速说明当前代码结构、接口现状、页面挂载情况、测试覆盖和下一步待开发清单。

## 1. 当前结论

- 当前仓库唯一活跃项目：`gongkao-project-v2/`
- 当前线上服务器：`192.168.1.15`
- 当前线上前端：`http://192.168.1.15:8888`
- 当前线上健康检查：`http://192.168.1.15:8001/health`
- 当前默认运维入口：`docker-compose.yml + .env`

如果后续继续开发，默认都从这个目录开始，不再进入已归档项目。

## 2. 代码结构

### 2.1 顶层

关键入口文件：

- `backend/`
- `frontend/`
- `docs/`
- `docker-compose.yml`
- `docker-compose.prod.yml`
- `deploy.sh`
- `push-update.sh`

### 2.2 后端结构

后端技术栈：

- FastAPI
- SQLAlchemy 2.0
- PostgreSQL
- Alembic
- APScheduler
- Anthropic API

后端主入口：

- `backend/app/main.py`

后端目录职责：

- `app/routes/`
  - HTTP 接口层
- `app/services/`
  - 业务逻辑层
- `app/repositories/`
  - 数据访问封装
- `app/models/`
  - ORM 模型
- `app/schemas/`
  - Pydantic 请求/响应模型
- `app/middleware/`
  - 认证、限流、异常处理、请求日志
- `app/tasks/`
  - 爬虫调度、每日提醒任务
- `app/utils/`
  - 缓存、日志、安全、脱敏

前端主入口：

- `frontend/src/App.tsx`

前端目录职责：

## 3. 后端接口现状

### 3.1 应用级接口

- `GET /health`
- `POST /api/v1/tasks/daily-reminders`
- `POST /api/v1/tasks/exam-reminders`

### 3.2 业务接口分组

认证与用户：

- `/api/v1/auth`
  - 登录、当前用户、退出、刷新 token、改密、更新资料
- `/api/v1/users`
  - 用户 CRUD、重置密码

学员与教学：

- `/api/v1/students`
  - 列表、详情、CRUD、状态流转、批量导入、批量分配督学、批量改状态、学员报告
- `/api/v1/supervision-logs`
  - 督学日志 CRUD、提醒列表
- `/api/v1/courses`
  - 课程 CRUD
- `/api/v1/homework`
  - 作业与提交管理
- `/api/v1/checkins`
  - 打卡、统计、排行榜
- `/api/v1/attendances`
  - 考勤 CRUD、签到码生成、二维码、扫码提交
- `/api/v1/study-plans`
  - 学习计划 CRUD、任务 CRUD
- `/api/v1`
  - 课程录播相关：教师、科目、班型、班次、录播

题库与考试：

- `/api/v1/exams`
  - 试卷、题目、成绩、二维码提交、AI 解析、班级/学员分析
- `/api/v1`
  - 题库相关：`questions`、`workbooks`
- `/api/v1`
  - 错题本相关：`mistakes`、`reviews`
- `/api/v1`
  - 薄弱项相关：模块树、学员弱项、雷达图

岗位与招考信息：

- `/api/v1/positions`
  - 列表、详情、筛选项、统计、匹配、推荐、收藏、分析、导入、智能导入、报告导出、对比
- `/api/v1/recruitment-info`
  - 公告列表、筛选项、采集状态、AI 日志、采集日志、详情、手动采集、配置、AI 分析、登录

经营与系统：

- `/api/v1/analytics`
  - 概览、督学统计、趋势、财务趋势、学员增长
- `/api/v1/finance`
  - 财务 CRUD、summary
- `/api/v1/packages`
  - 套餐与套餐项
- `/api/v1/notifications`
  - 通知列表、未读数、标记已读、全部已读、创建
- `/api/v1/calendar`
  - 日历 CRUD、upcoming、AI 解析、确认导入
- `/api/v1/export`
  - 学员、财务、考勤导出
- `/api/v1/audit-logs`
  - 审计日志
- `/api/v1/recycle-bin`
  - 回收站、恢复、统计
- `/api/v1/upload`
  - 文件上传

### 3.3 接口层观察

- 接口分层已经成型，模块拆分明确。
- 大部分模块都具备完整 CRUD 或接近完整 CRUD。
- 路由风格还不完全统一：
  - 有的模块前缀是 `/api/v1/<module>`
  - 有的模块前缀是 `/api/v1`，再在子路径中区分资源
- 文档里仍保留“统一 API 响应格式”“异常处理统一”等技术债。

## 6. 当前技术债与风险点

### 6.1 已在路线图中明确的技术债

来自 `docs/FEATURE_ROADMAP.md`：

- 统一 API 响应格式
- 后端异常处理统一
- 学员列表虚拟滚动
- 图表懒加载
- 敏感操作二次确认
- 审计日志完善

### 6.2 从代码结构直接看到的缺口

- 表单页和详情操作页并未完全接到路由
  - 学员新增/编辑
  - 课程新增/编辑
  - 督学日志新增/编辑
  - 提醒列表
- 前端测试基本空缺
- 路由组织方式还存在历史差异
  - `/api/v1/<module>` 和 `/api/v1 + 子路径` 混用
- 顶层存在较多历史文档和早期规划，真实状态要以代码为准

### 6.3 文档与实现的关系

- 旧路线图文档偏“完成汇总”与“功能蓝图”
- 当前真正可靠的开发真相，应以：
  - `backend/app/main.py`
  - `backend/app/routes/*`
  - `frontend/src/App.tsx`
  - `frontend/src/components/layout/MainLayout.tsx`
  为准

## 7. 待开发清单

- 保持 `gongkao-project-v2/` 作为唯一开发入口
- 所有改动优先补文档引用到这一份开发起点文档
- 新功能默认补对应测试，不再继续裸开发

   - `StudentForm`
   - `CourseForm`

## 8. 推荐开工顺序

如果现在开始继续开发，建议顺序：

1. 先接通未挂载页面，完成最小 CRUD 闭环
2. 然后补前端测试骨架
3. 再统一 API 响应与异常处理
4. 最后做性能和体验优化
