# 公考管理系统 - 用户使用手册

## 快速开始

### 登录系统
1. 浏览器打开 http://192.168.1.15
2. 输入用户名和密码
3. 点击「登录」

### 默认账号
| 角色 | 用户名 | 密码 | 权限 |
|------|--------|------|------|
| 管理员 | admin | admin123 | 全部功能 |
| 督学老师 | supervisor1 | 123456 | 学员管理、督学、教学 |

> 首次登录后请在「个人中心」修改密码。

## 功能模块说明

### 1. 数据看板（首页）
登录后的首页，展示核心运营数据：
- **核心指标**：学员总数、今日打卡、待跟进学员、学习计划
- **趋势图**：打卡/新增学员趋势（可切换7/14/30天）
- **学员分布**：各状态学员占比
- **考试倒计时**：近期考试提醒
- **快捷操作**：录入打卡、添加日志等常用入口

### 2. 学员管理

#### 学员列表
- **搜索**：按姓名或手机号搜索
- **筛选**：按状态（在读/试听/线索/休学/结业/退出）筛选
- **新建学员**：点击「新建学员」按钮，填写基本信息
- **导入**：支持 Excel 批量导入学员
- **导出**：导出学员数据为 Excel

#### 学员详情
点击学员姓名进入详情页：
- **基本信息**：查看/编辑学员资料
- **督学日志**：该学员的所有跟进记录
- **考勤记录**：出勤情况统计
- **考试成绩**：模考成绩和趋势
- **薄弱分析**：知识点掌握度雷达图

#### 学员状态说明
| 状态 | 颜色 | 含义 |
|------|------|------|
| 线索 | 灰色 | 潜在学员，未正式报名 |
| 试听 | 紫色 | 试听阶段 |
| 在读 | 绿色 | 正式学员，正在学习 |
| 休学 | 橙�� | 暂停学习 |
| 结业 | 蓝色 | 已完成学习 |
| 退出 | 红色 | 已退出 |

### 3. 督学日志
记录每次与学员的沟通和跟进：
- **新建日志**：选择学员 → 填写联系方式、心情、学习状态、内容
- **筛选**：按学员、日期范围、联系方式筛选
- **下次跟进**：设置下次联系日期，到期在首页提醒

### 4. 考试日历
日历视图管理各类事件：
- **事件类型**：考试、课程、模考、任务、自定义
- **添加事件**：点击日期添加，填写标题、类型、描述
- **提醒**：设置提前提醒天数
- **AI 添加**：粘贴文字，AI 自动解析为事件

### 5. 教学管理

#### 课程管理
- 管理培训课程（国省考/事业编等）
- 记录课程名称、教师、日期范围、状态

#### 作业管理
- 发布作业，关联课程
- 查看学员提交情况和批改状态

#### 班次管理
- 管理培训班次信息

#### 课程录播
- 管理录播视频链接

### 6. 题库模考

#### 试卷管理
- 创建和管理模考试卷
- 生成试卷二维码，学员扫码提交答案

#### 模考成绩
- 录入学员模考成绩
- 查看成绩统计和排名

#### 错题本
- 管理学员错题，标记掌握状态

#### AI 导入
- 上传试卷文档（PDF/Word）
- AI 自动解析并导入题目

### 7. 数据分析

#### 班级分析
- 选择试卷，查看班级整体表现
- 柱状图排名、雷达图知识点分析

#### 智能选岗
- 公务员岗位库查询
- 根据学员条件推荐匹配岗位

### 8. 日常管理

#### 打卡管理
- 查看学员每日学习打卡记录
- 统计连续打卡天数

#### 考勤管理
- 记录学员出勤情况
- 生成签到二维码

#### 套餐管理
- 管理学习套餐和定价

#### 财务管理
- 记录收支明细
- 查看财务统计

### 9. 系统管理（仅管理员）

#### 用户管理
- 添加/编辑督学老师账号
- 重置密码

#### 审计日志
- 查看所有操作记录

#### 回收站
- 恢复误删的数据

## 常用操作流程

### 新学员入学
1. 学员管理 → 新建学员 → 填写信息 → 状态设为「在读」
2. 分配督学老师
3. 创建学习计划
4. 添加到对应课程班次

### 每日督学工作
1. 查看首页「待跟进学员」
2. 联系学员后，添加督学日志
3. 检查学员打卡情况
4. 更新学习计划进度

### 模考流程
1. 试卷管理 → 新建试卷 → 添加题目
2. 生成二维码，学员扫码答题
3. 模考成绩 → 查看成绩统计
4. 班级分析 → 分析薄弱点

### 数据导出
- 学员列表页 → 导出按钮 → 下载 Excel
- 成绩页 → 导出成绩单

---

## 注意事项
- 删除数据为「软删除」，可在回收站恢复
- 重要操作会记录审计日志
- 建议定期修改密码
- 数据每天凌晨自动备份
