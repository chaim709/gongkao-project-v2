根据 v1 vs v2 数据库对比，v2 缺少以下核心功能：

## 第一阶段：核心功能补充（高优先级）

### 1. 学习计划系统 ⭐⭐⭐

**功能描述**：
- 为学员制定阶段性学习计划
- 计划模板管理
- 任务分解与进度跟踪
- AI 智能建议

**数据库设计**：
```sql
-- 学习计划表
study_plans (
  id, student_id, name, phase,
  start_date, end_date, status,
  ai_suggestion, notes,
  created_by, created_at, updated_at,
  deleted_at, deleted_by
)

-- 计划模板表
plan_templates (
  id, name, phase, duration_days,
  description, template_data,
  created_by, created_at, updated_at,
  deleted_at, deleted_by
)

-- 计划任务表
plan_tasks (
  id, plan_id, title, description,
  task_type, target_value, actual_value,
  due_date, status, priority,
  created_at, updated_at,
  deleted_at, deleted_by
)

-- 计划目标表
plan_goals (
  id, plan_id, goal_type, target_value,
  current_value, status,
  created_at, updated_at
)

-- 计划进度表
plan_progress (
  id, plan_id, task_id,
  progress_date, progress_value,
  notes, created_at
)
```

**API 设计**：
- GET /api/v1/study-plans - 学习计划列表
- POST /api/v1/study-plans - 创建学习计划
- GET /api/v1/study-plans/{id} - 学习计划详情
- PUT /api/v1/study-plans/{id} - 更新学习计划
- DELETE /api/v1/study-plans/{id} - 删除学习计划
- GET /api/v1/plan-templates - 计划模板列表
- POST /api/v1/study-plans/{id}/tasks - 添加任务
- PUT /api/v1/study-plans/tasks/{id}/progress - 更新任务进度

**前端页面**：
- 学习计划列表页
- 学习计划创建/编辑页
- 学习计划详情页（含任务列表、进度图表）
- 计划模板管理页

**工作量估算**：5-7 天
- 后端开发：3 天
- 前端开发：3 天
- 测试：1 天

**功能描述**：
- 课程录播视频管理
- 班次管理
- 课程表管理
- 教师管理
- 科目管理

**数据库设计**：
```sql
-- 教师表
teachers (
  id, name, phone, wechat,
  subject_ids, title, bio,
  status, created_at, updated_at,
  deleted_at, deleted_by
)

-- 科目表
subjects (
  id, name, code, category,
  description, sort_order,
  created_at, updated_at
)

-- 班次表
class_batches (
  id, name, class_type_id,
  start_date, end_date,
  teacher_id, student_count,
  status, description,
  created_at, updated_at,
  deleted_at, deleted_by
)

-- 班型表
class_types (
  id, name, description,
  duration_days, price,
  status, created_at, updated_at
)

-- 课程表
schedules (
  id, batch_id, subject_id, teacher_id,
  schedule_date, start_time, end_time,
  classroom, status, notes,
  created_at, updated_at,
  deleted_at, deleted_by
)

-- 课程录播表
course_recordings (
  id, batch_id, schedule_id,
  recording_date, period, title,
  recording_url, subject_id, teacher_id,
  duration_minutes, remark,
  created_at, created_by, updated_at,

schedule_change_logs (
```

**API 设计**：
- GET /api/v1/teachers - 教师列表
- GET /api/v1/subjects - 科目列表
- GET /api/v1/class-batches - 班次列表
- GET /api/v1/schedules - 课程表
- GET /api/v1/course-recordings - 录播列表
- POST /api/v1/course-recordings - 上传录播

**前端页面**：
- 教师管理页
- 班次管理页
- 课程表页（日历视图）
- 录播管理页（视频列表、播放器）

**工作量估算**：7-10 天
- 后端开发：4 天
- 前端开发：4 天

- 测试：1 天

## 技术要点

### 后端
- 继续使用 FastAPI + PostgreSQL
- 遵循现有分层架构
- 所有表支持软删除
- 关键操作记录审计日志

### 前端
- 继续使用 React + TypeScript
- 视频播放器：Video.js 或 DPlayer
- 日历组件：FullCalendar
- 图表组件：Recharts

### 小程序
- 微信小程序原生开发或 Taro
- 视频播放：微信原生 video 组件
- 状态管理：MobX 或 Zustand
