# 公考培训机构管理系统 - 功能规划与开发路线图

> 基于实际运营需求讨论整理，覆盖教学管理、招生获客、数据分析、财务管理等全业务线。

---

## 一、系统定位

面向**线下公务员/事业编考试培训机构**的一站式管理系统，核心目标：

1. **提升教学效果** — 通过错题分析、弱项诊断、智能推题提高学员通过率
2. **数据驱动决策** — 通过率统计、经营数据看板辅助机构运营
3. **降低管理成本** — 签到、财务、学员管理自动化
4. **增强招生能力** — 用数据化教学成果作为招生武器

**数据库表设计：**

```sql
-- 试卷/练习册
CREATE TABLE exam_papers (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,          -- "2025年江苏省考行测模拟卷（第3期）"
    exam_type VARCHAR(50),                -- 国考/省考/事业单位
    subject VARCHAR(50) NOT NULL,         -- 行测/申论/公基/职测
    total_questions INTEGER NOT NULL,     -- 总题数
    time_limit INTEGER,                   -- 考试时间（分钟）
    year INTEGER,                         -- 年份
    source VARCHAR(100),                  -- 来源（自编/真题/模拟）
    qr_code_url VARCHAR(500),            -- 二维码链接
    created_by INTEGER REFERENCES users(id),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    deleted_at TIMESTAMPTZ
);

-- 题目
CREATE TABLE questions (
    id SERIAL PRIMARY KEY,
    paper_id INTEGER REFERENCES exam_papers(id),
    question_number INTEGER NOT NULL,     -- 在试卷中的题号
    question_type VARCHAR(30) NOT NULL,   -- single_choice/multi_choice/essay/fill
    content TEXT,                         -- 题干文字（文字题）
    content_image_url VARCHAR(500),       -- 题干图片（图形推理等）
    options JSONB,                        -- {"A":"xxx","B":"xxx","C":"xxx","D":"xxx"} 或图片URL
    correct_answer VARCHAR(20) NOT NULL,  -- B / AB / 主观题为空
    explanation TEXT,                     -- 解析
    difficulty VARCHAR(20),               -- easy/medium/hard
    knowledge_category VARCHAR(50),       -- 一级分类：言语理解与表达
    knowledge_tag VARCHAR(100),           -- 二级分类：主旨概括
    knowledge_detail VARCHAR(200),        -- 三级分类（可选）
    key_technique VARCHAR(200),           -- 解题技巧
    common_mistake VARCHAR(200),          -- 易错点
    source VARCHAR(50),                   -- manual/ai_import
    ai_confidence FLOAT,                 -- AI识别置信度
    created_at TIMESTAMPTZ DEFAULT NOW(),
    deleted_at TIMESTAMPTZ
);

-- 学员答题记录
CREATE TABLE student_answers (
    id SERIAL PRIMARY KEY,
    student_id INTEGER REFERENCES students(id) NOT NULL,
    paper_id INTEGER REFERENCES exam_papers(id) NOT NULL,
    question_id INTEGER REFERENCES questions(id) NOT NULL,
    student_answer VARCHAR(20),           -- 学生的答案
    is_correct BOOLEAN NOT NULL,
    time_spent INTEGER,                   -- 答题用时（秒，在线答题模式）
    submitted_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(student_id, question_id, submitted_at::date)
);

-- 错题本
CREATE TABLE mistake_records (
    id SERIAL PRIMARY KEY,
    student_id INTEGER REFERENCES students(id) NOT NULL,
    question_id INTEGER REFERENCES questions(id) NOT NULL,
    wrong_count INTEGER DEFAULT 1,        -- 错误次数
    last_wrong_at TIMESTAMPTZ,
    is_mastered BOOLEAN DEFAULT FALSE,    -- 是否已掌握（重做正确）
    mastered_at TIMESTAMPTZ,
    note TEXT,                            -- 学生备注
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 模考成绩
CREATE TABLE exam_scores (
    id SERIAL PRIMARY KEY,
    student_id INTEGER REFERENCES students(id) NOT NULL,
    paper_id INTEGER REFERENCES exam_papers(id) NOT NULL,
    total_score FLOAT,                    -- 总分
    correct_count INTEGER,                -- 正确题数
    wrong_count INTEGER,                  -- 错误题数
    blank_count INTEGER,                  -- 未答题数
    time_used INTEGER,                    -- 用时（分钟）
    score_detail JSONB,                   -- 分模块得分 {"言语":20,"判断":15,...}
    rank_in_class INTEGER,                -- 班级排名
    submitted_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 3.2 AI文档解析入库

**流程：**
```
老师上传 PDF/Word
      ↓
后端接收文件，调用AI API
      ↓
AI识别并返回结构化数据：
  - 题号、题干（文字/图片）
  - 选项A/B/C/D（文字/图片）
  - 正确答案
  - 知识点分类
  - 解析、易错点
      ↓
前端展示识别预览
  - 老师审核、修正
  - 支持单题编辑
      ↓
确认入库
  - 写入questions表
  - 自动生成试卷二维码
```

**技术方案：**
- 后端：FastAPI接收文件上传 → 调用Claude API（支持PDF视觉识别）
- 图片处理：AI从PDF中提取图形推理图片，存储到文件系统/OSS
- 前端：上传组件 + 识别结果预览表格 + 编辑弹窗

**AI识别能力：**
- 文字题：提取题干、选项、答案（准确率极高）
- 图形推理：识别图片并标记为图形题类型，保存原图
- 知识点分类：根据题目内容自动分类到对应知识点标签
- 解析生成：自动生成解题思路和易错点

**题目类型处理策略：**

| 题目类型 | 处理方式 |
|----------|----------|
| 文字选择题（言语、逻辑、常识） | AI全自动提取文字 |
| 图形推理 | AI提取图片 + 自动分类 |
| 资料分析（含表格/图表） | AI提取表格数据 + 题目文字 |
| 申论/主观题 | AI提取材料和题目，答案留空或参考答案 |
| 数学公式题 | AI提取文字，公式用LaTeX或图片 |

#### 3.3 二维码错题提交（H5页面）

**学生端流程：**
```
扫描试卷上的二维码（微信/浏览器）
      ↓
打开H5页面（不需要APP/小程序）
      ↓
输入姓名或手机号识别身份
      ↓
两种方式提交错题：
  方式一：手动输入 "3, 15, 22, 47"
  方式二：点选题号网格（点击变红）
      ↓
提交 → 系统自动记录
```

**页面设计要点：**
- 移动端优先，大按钮大字体
- 题号网格：每行10个数字按钮，点击切换红色/灰色
- 支持记住身份（下次扫码自动识别）
- 提交后显示简要反馈："本次错了8题，主要集中在数量关系"

**技术方案：**
- 独立的H5页面，不需要登录管理系统
- 通过试卷二维码中的paper_id关联试卷
- 学生通过手机号识别身份
- 提交后自动写入student_answers和mistake_records

#### 3.4 弱项分析与智能推题

**学员能力画像：**
```
张三 - 能力雷达图
        言语 ████████░░ 80%
      判断推理 ██████░░░░ 60%
      数量关系 ███░░░░░░░ 35%  ← 最弱
      资料分析 ████████░░ 82%
      常识判断 ██████░░░░ 65%

最近4周趋势：数量关系 25% → 30% → 32% → 35% ↑缓慢提升
薄弱知识点：排列组合(错5次)、工程问题(错4次)、概率(错3次)
建议：重点攻克排列组合，建议每天练习3-5题
```

**系统自动分析维度：**
- 按科目统计正确率（言语、判断、数量、资料、常识）
- 按知识点统计正确率（细化到二级标签）
- 错题频率分析（哪些知识点反复错）
- 进步趋势（周/月正确率变化）
- 与同班同学对比排名

**智能推题逻辑：**
- 优先推送错误率最高的知识点题目
- 隔7天自动推送错题重做（检验是否掌握）
- 重做正确则标记为"已掌握"
- 重做仍错则加权推送更多同类题目

#### 3.5 模考成绩追踪

**功能：**
- 每次模考成绩录入（总分、分模块得分）
- 自动生成成绩趋势图（折线图）
- 班级排名变化
- 分模块得分对比（柱状图）
- 与历史数据对比

**老师看到的视角：**
- 班级整体：这套卷子全班正确率最低的10题（需要重讲）
- 个人档案：某学员连续4次模考数量关系低于40%
- 进步之星：本月进步最大的5名学员
- 预警学员：成绩持续下滑的学员

**试听课管理：**
- 试听学员单独建档
- 试听后3天未跟进自动告警
- 试听转化率统计

### 4.2 学员全生命周期

```
线索 → 试听 → 报名 → 在读 → 考试 → 上岸/未上岸 → 校友
```

**考试结果管理：**
- 记录每个学员参加的考试
- 笔试分、面试分、是否进面、是否录取
- 自动计算通过率

**未上岸学员续费：**
- 没考上的学员自动标记

**校友管理：**
- 上岸学员记录去向

## 七、关键设计原则

1. **学生操作成本最低化** — 扫码 + 输入几个数字就够了，不要让学生下载APP
2. **老师操作简单化** — AI做80%的工作，老师只审核确认
3. **数据先行** — 先跑起来收集数据，再逐步完善分析功能
4. **分阶段交付** — 每个Phase独立可用，不需要全做完才能用
5. **移动端友好** — 签到、错题提交等高频操作必须手机可用
