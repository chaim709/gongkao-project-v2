# 公考培训机构管理系统 - 前端 UI 设计文档

## 一、项目概述

### 产品定位
公务员考试培训机构内部管理系统（B 端后台），面向培训机构管理员和督学老师使用。

### 核心用户
- **管理员**：机构负责人，管理全部功能
- **督学老师**：负责学员跟进、学习计划制定、成绩录入

### 技术栈
- React 18 + TypeScript
- Ant Design 5.x 组件库
- Tailwind CSS 辅助样式
- Recharts 图表库
- React Router 路由
- TanStack Query 数据管理

### 当前问题
1. 纯用 Ant Design 默认样式，没有品牌感和设计感
2. 所有页面风格雷同（搜索栏 + 表格），视觉单调
3. 仪表盘信息过载，重点不突出
4. 登录页过于简陋
5. 颜色、间距、排版缺乏统一设计规范
6. 侧边栏菜单项过多（8 个分组、30+ 子项），导航体验差

## 三、各页面详细说明

### 1. 登录页 `/login`

**功能**：用户名 + 密码登录

**当前设计**：灰色背景 + 白色卡片居中，非常简陋

**期望**：
- 专业感的登录页面，体现"公考培训"定位
- 左右分栏或全屏背景 + 毛玻璃卡片
- 展示产品名称和简介
- 表单：用户名输入框（带用户图标）、密码输入框（带锁图标）、登录按钮

### 2. 数据看板 `/dashboard`（重点设计）

**角色**：系统首页，管理员和督学老师每天第一个看到的页面

**当前问题**：
- 塞了太多图表（8 个区块），信息过载
- 包含"学员心情分布"等无用数据
- 图表太小太挤

**需要展示的数据（重新规划后）**：

**第一行 - 核心指标卡片（4 个）**：

| 指标 | 主数据 | 副数据 |
|------|--------|--------|
| 学员总数 | 在读学员数 / 总学员数 | 本月新增 +N |
| 今日打卡 | 已打卡人数 / 应打卡人数 | 打卡率 XX% |
| 待跟进学员 | N 人 | 超�� 7 天未联系 |
| 学习计划 | 进行中 N 个 | 完成率 XX% |

**第二行 - 趋势 + 分布（左右两列）**：
- 左侧（大）：折线趋势图（打卡数、新增学员数，最近 7/14/30 天可切换）
- 右侧（小）：学员状态分布环形图（在读/试听/线索/休学/结业/退出）

**第三行 - 考试倒计时 + 待办（左右两列）**：
- 左侧：考试倒计时列表（考试名称 + 倒计时天数 + 紧急度标签，最多 5 条）
- 右侧：待办事项区域
  - 待跟进学员（3-5 条：学员名 + 最后联系时间）
  - 近期学习计划（3 条：计划名 + 进度条）
  - 快捷操作按钮（录入打卡 / 添加日志 / 新建计划 / 导入成绩）

**第四行（管理员可见）- 运营数据**：
- 左侧：本月财务摘要（收入数字 + 同比环比）
- 右侧：考试统计摘要（模考次数、平均正确率、最薄弱知识点 Top 3）

## 四、设计要求

### 1. 整体风格
- 现代、简洁、专业的 B 端后台风格
- 适合教育培训行业的视觉调性
- 参考风格：飞书后台、语雀管理后台、钉钉工作台
- 不要过于花哨，注重信息层次和可读性

### 2. 色彩体系
- 需要一套完整的色彩方案（主色、辅助色、状态色、中性色）
- 主色建议偏蓝/靛蓝色系（专业、信任感）
- 状态色：成功(绿)、警告(橙)、错误(红)、信息(蓝)
- 学员状态需要 6 种可区分的颜色

### 3. 关键设计页面（优先级排序）

**P0 必须设计**：
1. 数据看板（仪表盘）- 最重要
2. 登录页
3. 整体布局框架（侧边栏 + 头部 + 内容区）

**P1 重要**：
4. 学员列表页（代表所有列表页的设计规范）
5. 学员详情页（代表详情页的设计规范）

**P2 可选**：
6. 考试日历页
7. 移动端打卡/提交页

### 4. 组件规范
- 基于 Ant Design 5.x，通过 ConfigProvider theme 自定义
- 需要定义：卡片样式、表格样式、搜索栏样式、按钮样式、标签样式
- 统一的圆角、阴影、间距规范

### 5. 响应式
- 最小支持宽度 1280px
- 侧边栏可折叠
- 移动端页面（打卡/提交）需要适配手机屏幕
